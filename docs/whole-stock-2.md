# Whole Stock 2 전종목 수급 스코어링 시스템

> `전종목_수급.xlsx`에 정리된 가격·수익률·거래대금·기관/외국인 수급 데이터를 읽어 전종목 투자 매력도를 점수화하는 Bloomberg 미사용 내부 분석 도구

---

## 개요

`whole-stock-2.py`는 국내 전종목 수급 데이터 파일을 기반으로, 매일 장 마감 후 종목별 가격 모멘텀, 수급 강도, 거래대금 증가 여부를 종합 평가하여 Excel 리포트를 생성합니다.

본 도구는 Bloomberg Terminal 또는 Bloomberg API를 직접 호출하지 않습니다. 가격·수익률·수급 데이터는 `전종목_수급.xlsx`(데이터가이드 export)를 사용하고, 장마감 후 당일 종가·거래량은 FinanceDataReader로 별도 조회하여 보강합니다.

> **가격 모멘텀의 핵심 입력은 엑셀의 `1주전대비수익률`·`1개월전대비수익률` 컬럼입니다.** 엑셀은 거래일이 ~20일뿐이라 종가만으로 20일 모멘텀을 계산하면 전종목 결측이 되므로, 데이터가이드가 서버측에서 산출한 수익률 컬럼을 직접 읽어 사용합니다.

## 실행 방법

`전종목_수급.xlsx`를 **이 스크립트와 같은 폴더**에 두고(레포에 푸시), 장마감 후 아래 명령어로 실행합니다.

```
py -3.12 whole-stock-2.py
```

실행 전 체크리스트:

- [ ] 스크립트와 같은 폴더에 `전종목_수급.xlsx` 존재 확인
- [ ] `Sheet1`에 최신 종가·수익률·거래대금·수급 데이터 반영
- [ ] 실행 로그에 `평균수익률 컬럼: 사용`, `거래대금: 로드됨 / 수급: 로드됨` 표시 확인
- [ ] 장마감 후 FinanceDataReader에서 당일 OHLCV가 제공되는 시간인지 확인
- [ ] 출력 폴더 `[오후] whole-stock-2` 쓰기 권한 확인
- [ ] 같은 날짜 결과 파일이 Excel에서 열려 있지 않은지 확인

> 경로는 하드코딩되어 있지 않습니다. `BASE_DIR = Path(__file__).resolve().parent` 로 **스크립트 위치 기준**이라 10845 머신/다른 머신 어디서 실행하든 같은 폴더의 입력 파일을 읽고 같은 폴더에 결과를 저장합니다.

```
<스크립트 폴더>/전종목_수급.xlsx
```

### ② 입력 파일 구조

`전종목_수급.xlsx`는 종목별 데이터가 열 방향으로 반복되는 와이드 구조입니다(데이터가이드 Refresh export).

| 행(1-based) | 내용 |
|----|------|
| 9행 | 종목 코드 (`A005930` 형식) |
| 10행 | 종목명 |
| 12행 | 항목 코드 |
| 13행 | 항목명 |
| 15행 이후 | 날짜별 데이터 |

사용 항목:

| 항목명(엑셀 라벨 예시) | 내부 컬럼 | 설명 |
|--------|-----------|------|
| `종가(원)` | `close` | 일별 종가 |
| `1주전대비수익률(%)` | `ret_5d` (`avg_ret_5d`) | 1주 전 대비 수익률(단기 모멘텀) |
| `1개월전대비수익률(%)` | `ret_20d` (`avg_ret_20d`) | 1개월 전 대비 수익률(중기 모멘텀) |
| `5일평균거래대금(원)` | `turnover_avg_5d` | 최근 5일 평균 거래대금 |
| `20일평균거래대금(원)` | `turnover_avg_20d` | 최근 20일 평균 거래대금 |
| `5일누적 기관 순매수대금(일간)(만원)` | `inst_net_5d` | 기관 최근 5일 순매수 |
| `20일누적 기관 순매수대금(일간)(만원)` | `inst_net_20d` | 기관 최근 20일 순매수 |
| `5일누적 외국인총합계 순매수대금(일간)(만원)` | `foreign_net_5d` | 외국인 최근 5일 순매수 |
| `20일누적 외국인총합계 순매수대금(일간)(만원)` | `foreign_net_20d` | 외국인 최근 20일 순매수 |

> 수급 데이터는 만원 단위, 거래대금 데이터는 원 단위로 들어온다고 가정합니다.

#### 라벨 매핑 방식 (중요)

엑셀 항목 라벨은 데이터가이드 버전마다 표기가 조금씩 다릅니다(예: `5일평균거래대금(원)` vs `거래대금 (5일 평균)(원)`, `1주전대비수익률(%)` vs `5일평균수익률(%)`). 정확 문자열 매칭은 라벨이 바뀌면 해당 컬럼이 통째로 누락되므로, **키워드 + 기간 패턴 리졸버**(`resolve_item_field`)로 매핑합니다.

- `종가` 포함 → `close`
- `수익` 포함 → `주`면 단기(`avg_ret_5d`), `월`이면 중기(`avg_ret_20d`), 그 외 `N일` 패턴이면 5/20 슬롯
- `거래대금` 포함 → `N일` 기준 5/20 슬롯
- `순매수`+`기관` → `N일` 기준 5/20 슬롯
- `순매수`+`외국인` → `N일` 기준 5/20 슬롯

이 방식 덕분에 구·신 라벨과 수익률 컬럼 표기 변화에 모두 대응합니다.

### ③ 당일 가격·거래량 보강 (FinanceDataReader)

| 수집 방식 | 역할 | 비고 |
|-----------|------|------|
| FinanceDataReader | 장마감 후/장중 **당일 종가·거래량 + 가격 모멘텀 재계산** | 종목코드 6자리, 최근 80일 OHLCV 조회 |

FinanceDataReader에서 엑셀 기준일보다 최신 거래일이 확인되면, 해당 날짜의 종가·거래량·거래대금을 보강하고 **가격 모멘텀(1주/1개월 수익률, 이동평균, 변동성, 추세정렬)을 FDR 당일가 기준으로 다시 계산**합니다. 당일 거래대금은 아래로 계산합니다.

```
당일 거래대금 = 당일 종가 × 당일 거래량
```

> **FDR 종가와 엑셀 종가는 수정주가 기준이 동일함을 확인**(같은 거래일 종가 일치)했기에, FDR 당일가로 모멘텀을 재계산해도 엑셀 수익률과 일관됩니다. 이로써 엑셀 기준일(예: 전일) 이후 당일에 **고점에서 반전한 종목은 수익률이 즉시 낮아져 순위가 하락**합니다. 1주는 5거래일, 1개월은 20거래일로 근사합니다.
>
> FDR `DataReader`는 **일봉 종가**라 장중에는 현재 체결가 스냅샷(실시간 호가는 아님), 장마감 후에는 확정 종가입니다. FDR 최신 거래일이 엑셀 기준일보다 늦지 않으면 해당 종목은 엑셀 기준 데이터를 그대로 사용하고, 결과 파일의 `당일가격반영` 컬럼이 `False`로 표시됩니다.

## 파생 지표 계산

### 가격 모멘텀 지표

| 지표 | 내부 컬럼 | 계산 방식 | 의미 |
|------|-----------|-----------|------|
| 1주 수익률 | `ret_5d` | 엑셀 `1주전대비수익률`(FDR 보강 시 당일가로 재계산) | 단기 가격 모멘텀 |
| 1개월 수익률 | `ret_20d` | 엑셀 `1개월전대비수익률`(FDR 보강 시 당일가로 재계산) | 중기 가격 모멘텀 |
| 일간변동성(20일) | `vol_20d` | 종가 일간수익률의 20일 표준편차 × 100 | 추세의 변동성 |
| 위험조정 모멘텀 | `risk_adj_mom` | `ret_20d ÷ vol_20d` | 변동성 1단위당 추세 강도 |
| 5일선/10일선/20일선 | `ma5/ma10/ma20` | 종가 이동평균 | 추세 기준선 |
| 추세정렬 점수 | `trend_align_score` | 아래 tanh 식, 0~100 | 종가/이평 정렬 강도(연속) |

추세정렬 점수:

```
trend_align = 0.5 · tanh(8 · (종가/10일선 − 1)) + 0.5 · tanh(8 · (5일선/10일선 − 1))
추세정렬 점수 = (trend_align + 1) × 50      # 0~100, 종가>이평 & 단기>중기일수록 100에 근접
```

> 과거의 이진 지표(`10일선 상회 여부`, `5일선>10일선 여부`)는 거리(magnitude)를 반영하는 연속 점수(`추세정렬 점수`)로 대체되었습니다. 두 이진 플래그(`above_ma10`, `ma5_above_ma10`)는 참고용으로 결과 파일에 함께 출력됩니다.

### 수급 지표

| 지표 | 처리 |
|------|------|
| 기관/외국인 5일 순매수 비율 | 순매수(만원)×10,000 ÷ (5일평균거래대금(원) × 5) × 100 |
| 기관/외국인 20일 순매수 비율 | 순매수(만원)×10,000 ÷ (20일평균거래대금(원) × 20) × 100 |
| 기관/외국인 수급 양수 비율(20일) | 최근 20개 관측치 중 5일 순매수 비율이 양수였던 비율 |

수급 비율 예시:

```
기관 5일 순매수 비율 = 기관5일순매수(만원) × 10,000 ÷ (거래대금5일평균(원) × 5) × 100
외국인 20일 순매수 비율 = 외국인20일순매수(만원) × 10,000 ÷ (거래대금20일평균(원) × 20) × 100
```

### 거래대금 지표

| 지표 | 처리 |
|------|------|
| 당일 거래량/거래대금 | FDR 보강 시 당일 값, 미보강 시 결측 |
| 거래대금 RVOL | `turnover_rvol` = (FDR 보강 시) 당일 거래대금 / 20일 평균, (엑셀 기준) 5일 평균 / 20일 평균 |

## 종합 점수 산출 방식

세 가지 평가 영역을 가중 합산하여 0~100점 종합 점수를 부여합니다.

```
종합 점수 = 가격 (50%) + 수급 (20%) + 거래대금 (30%)
```

각 세부 항목은 전종목 횡단면 백분위 점수로 정규화합니다(`normalize_score`). 즉, 절대값 자체보다 같은 날 전체 종목 중 상대적으로 얼마나 강한지를 평가합니다. 결측값은 중립값 50점으로 처리합니다.

### 가격 점수 (50%) — 모멘텀 4축

서로 상관이 낮은 4개 축으로 구성됩니다(추세 / 추세의 질 / 단기추세 / 추세정렬).

| 항목 | 가중치 | 의미 |
|------|--------|------|
| 1개월 수익률 백분위 | 40% | 최근 한 달 가격 모멘텀(핵심) |
| 위험조정 모멘텀 백분위 | 25% | 변동성 대비 추세 강도(고변동성 편향 완화) |
| 1주 수익률 백분위 | 20% | 단기 가격 모멘텀 |
| 추세정렬 점수 | 15% | 종가/이동평균 정렬(연속값, 이미 0~100) |

산식:

```
가격 점수 =
    rank(1개월 수익률).clip(2,98)     × 0.40
  + rank(위험조정 모멘텀).clip(2,98)  × 0.25
  + rank(1주 수익률).clip(2,98)       × 0.20
  + 추세정렬 점수(fillna 50)          × 0.15
```

> 수익률·위험조정 모멘텀 백분위 점수는 극단값 영향을 줄이기 위해 2~98점 범위로 제한합니다.

### 수급 점수 (20%)

| 항목 | 가중치 | 의미 |
|------|--------|------|
| 외국인 5일 순매수 비율 | 25% | 단기 외국인 매수 강도 |
| 외국인 20일 순매수 비율 | 15% | 중기 외국인 매수 방향성 |
| 외국인 수급 양수 비율(20일) | 10% | 최근 20일간 외국인 수급 지속성 |
| 기관 5일 순매수 비율 | 25% | 단기 기관 매수 강도 |
| 기관 20일 순매수 비율 | 15% | 중기 기관 매수 방향성 |
| 기관 수급 양수 비율(20일) | 5% | 최근 20일간 기관 수급 지속성 |
| 기관·외국인 동시 순매수 | 5% | 양대 수급 주체 동시 유입 여부 |

수급은 단순 순매수 금액이 아니라 거래대금 대비 비율로 평가합니다. 대형주가 절대 금액만으로 과도하게 유리해지는 문제를 줄이기 위한 처리입니다.

### 거래대금 점수 (30%)

| 항목 | 가중치 | 의미 |
|------|--------|------|

## 내부 구현 상세

`whole-stock-2.py`는 실행 진입점만 담당합니다. 실제 데이터 파싱, 지표 계산, 스코어링, Excel 저장은 `score_all_supply.py`에서 처리합니다.

### ① 기본 설정

| 항목 | 값 | 설명 |
|------|----|------|
| 입력 파일 | `<스크립트 폴더>/전종목_수급.xlsx` | 원천 데이터 (`Path(__file__).parent` 기준) |
| 출력 폴더 | `<스크립트 폴더>/[오후] whole-stock-2` | 결과 저장 위치 |
| 사용 시트 | `Sheet1` | 우선적으로 읽는 시트 |
| 가격 가중치 `PRICE_WEIGHT` | 0.50 | 가격 모멘텀 반영 비중 |
| 수급 가중치 `SUPPLY_WEIGHT` | 0.20 | 기관·외국인 수급 반영 비중 |
| 거래대금 가중치 `VALUE_WEIGHT` | 0.30 | 거래대금 증가 및 유동성 반영 비중 |
| FDR 조회 기간 `FDR_LOOKBACK_DAYS` | 80일 | 당일 가격·거래량·거래대금 보강용 |

항목 라벨은 `resolve_item_field`(키워드+기간 리졸버)로 내부 컬럼명에 매핑됩니다. 예: `종가(원)`→`close`, `1개월전대비수익률(%)`→`avg_ret_20d`(→`ret_20d`), `5일평균거래대금(원)`→`turnover_avg_5d`.

### ② 점수 정규화 방식

모든 세부 점수는 같은 날짜의 전체 종목을 대상으로 백분위 순위(`rank(pct=True) × 100`)를 계산합니다.

- 서로 단위가 다른 지표를 0~100점 범위로 통일
- 절대 금액이 큰 대형주 편향 완화
- 매일 시장 전체 안에서 상대적으로 강한 종목을 선별

결측값은 50점(중립)으로 처리합니다.

### ③ 입력 시트 선택

기본적으로 `Sheet1`을 사용합니다. 없으면 9행에 `A` + 6자리 종목코드가 가장 많은 시트를 자동 선택합니다(운영 기준은 `Sheet1` 고정).

### ④ 원천 데이터 파싱

1. 9행 종목 코드 → 10행 종목명 → 13행 항목명 추출
2. 항목명을 `resolve_item_field`로 내부 필드에 매핑(매핑 실패 열은 제외)
3. 15행 이후 날짜별 값을 숫자로 변환
4. (날짜·코드·항목·값) 긴 테이블로 만든 뒤 (날짜·종목) 분석용 테이블로 피벗

로드 후 콘솔에 로드된 필드 목록과 `평균수익률 컬럼 / 거래대금 / 수급` 적재 여부를 출력합니다.

### ⑤ 시계열 파생 지표 계산 (`add_time_series_features`)

종목별 날짜순 정렬 후:

- 이동평균 `ma5/ma10/ma20` (rolling, 최소관측 3/5/10)
- `ret_5d`/`ret_20d`: 엑셀 수익률 컬럼 우선, 없으면 종가 pct_change 폴백
- `vol_20d`: 종가 일간수익률의 20일 표준편차 × 100
- `risk_adj_mom = ret_20d / vol_20d`
- `trend_align_score`: 종가/이평 tanh 정렬 점수(0~100)
- 기관/외국인 5·20일 순매수 비율, 수급 양수 비율(20일)
- `turnover_rvol = turnover_avg_5d / turnover_avg_20d`

### ⑥ FinanceDataReader 가격 보강 (`apply_fdr_price_overlay`)

처리 순서:

1. 엑셀에서 파싱한 종목코드별 최신 행 목록 생성
2. 종목별 최근 80일 OHLCV를 FinanceDataReader로 조회
3. FDR 최신 거래일이 엑셀 기준일보다 최신이면, 엑셀 최신 행을 복사해 새 가격 행 생성
4. 새 행의 **종가·거래량·거래대금·거래대금 평균/RVOL**을 FDR 기준으로 갱신
5. **1주/1개월 수익률·이동평균·변동성·추세정렬을 FDR 당일가 기준으로 재계산**
6. 새 거래대금으로 수급 비율 재계산 (수급 순매수 원본은 엑셀 값 유지)

이 구조로 결과 파일에는 기준일이 두 개로 표시됩니다.

| 기준일 | 의미 |
|--------|------|
| 수급 기준일 | `전종목_수급.xlsx`의 최신 날짜 |
| 가격/거래량 기준일 | FinanceDataReader에서 확인한 최신 거래일(보강 시) |

FDR 최신 거래일이 당일이면 `당일가격반영=True`. FDR 데이터가 늦거나 누락된 종목은 엑셀 기준 데이터를 사용하고 `당일가격반영=False`로 표시됩니다.

### ⑦ 기준일 선정

스코어링은 종목별 최신 행(보강 시 FDR 가격 행)을 대상으로 수행합니다. 보강 시 가격/거래량과 **가격 모멘텀은 FDR 당일가 기준**, 수급 순매수는 엑셀 기준일 값을 사용합니다.

### ⑧ 스코어 계산 순서

기준일 데이터에서 플래그를 먼저 만듭니다.

| 플래그 | 의미 |
|--------|------|
| `10일선상회` | 기준일 종가가 10일선보다 높은지(참고용) |
| `5일선>10일선` | 5일선이 10일선보다 높은지(참고용) |
| `기관외인동시순매수` | 기관·외국인 5일 순매수 비율이 모두 양수인지 |

이후 가격·수급·거래대금 스코어를 각각 계산하고 종합스코어를 산출합니다.

```
종합스코어 = 가격스코어 × 0.50 + 수급스코어 × 0.20 + 거래대금스코어 × 0.30
```

등급은 종합스코어 구간(`30/50/65/80`)으로 A~F를 부여합니다.

### ⑨ 결과 파일 생성

Excel 파일 하나로 저장하며, 첫 번째 시트에 보고서 형태의 요약 화면을 생성합니다.

보고서 시트 구성: 제목·기준일, 전체/A등급/65점↑/80점↑/당일가격반영 종목 수, 추천종목 TOP 테이블, A+B/C/D+F 등급 분포 차트.

데이터 시트에는 필터·고정 헤더·컬럼 폭·테두리/색상을 적용합니다.

### ⑩ 콘솔 출력

실행이 끝나면 TOP 20 종목(date, code, name, total_score, grade, price_score, supply_score, value_score)과 저장된 Excel 경로를 출력합니다.

## 운영 유의사항

- 이 도구는 Bloomberg Terminal을 사용하지 않습니다. 입력 파일 `전종목_수급.xlsx`가 최신화되어 있어야 합니다.
- **가격 모멘텀은 엑셀 수익률 컬럼이 핵심**이며, FDR이 더 최신 거래일을 제공하면 당일가로 재계산됩니다. `1주전대비수익률`·`1개월전대비수익률`이 누락되고 FDR도 실패하면 종가 pct_change로 폴백하는데, 엑셀 거래일이 ~20일이라 1개월 모멘텀이 결측이 될 수 있습니다.
- FDR 당일가 기준 모멘텀은 1주=5거래일, 1개월=20거래일 근사이며, FDR `DataReader`는 일봉 종가라 장중에는 실시간 호가가 아닌 현재 체결가 스냅샷입니다.
- 항목 라벨이 바뀌어도 리졸버가 키워드로 매핑하지만, 키워드(종가/수익/거래대금/순매수/기관/외국인)나 기간 표기(주/월/N일)가 크게 달라지면 `resolve_item_field`를 수정해야 합니다.
- 현재는 `Sheet1`을 고정으로 읽습니다. 시트명이 바뀌면 `SOURCE_SHEET`를 수정해야 합니다.
- 수급 데이터는 만원 단위, 거래대금은 원 단위로 계산합니다. 입력 파일 단위가 바뀌면 수급 비율이 왜곡됩니다.
- 종합스코어는 같은 날 종목 간 상대 백분위 기반입니다. 날짜가 다른 결과 파일끼리 직접 비교할 때는 시장 환경 차이를 고려해야 합니다.
- 결과 파일이 Excel에서 열려 있으면 저장 실패가 발생할 수 있습니다.
