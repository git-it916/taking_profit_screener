# Whole Stock 2 전종목 수급 스코어링 시스템

> `전종목_수급.xlsx`에 이미 정리된 가격·거래대금·기관/외국인 수급 데이터를 읽어 전종목 투자 매력도를 점수화하는 Bloomberg 미사용 내부 분석 도구

---

## 개요

`whole-stock-2.py`는 국내 전종목 수급 데이터 파일을 기반으로, 매일 장 마감 후 종목별 가격 모멘텀, 수급 강도, 거래대금 증가 여부를 종합 평가하여 Excel 리포트를 생성합니다.

본 도구는 Bloomberg Terminal 또는 Bloomberg API를 직접 호출하지 않습니다. 수급 데이터는 `전종목_수급.xlsx`를 사용하고, 장마감 후 당일 종가·거래량은 FinanceDataReader로 별도 조회하여 반영합니다.

### ② 입력 파일 구조

`전종목_수급.xlsx`는 종목별 데이터가 열 방향으로 반복되는 와이드 구조입니다.

| 행 | 내용 |
|----|------|
| 9행 | 종목 코드 (`A005930` 형식) |
| 10행 | 종목명 |
| 12행 | 항목 코드 |
| 13행 | 항목명 |
| 15행 이후 | 날짜별 데이터 |

사용 항목:

| 항목명 | 내부 컬럼 | 설명 |
|--------|-----------|------|
| `종가(원)` | `close` | 일별 종가 |
| `거래대금 (5일 평균)(원)` | `turnover_avg_5d` | 최근 5일 평균 거래대금 |
| `거래대금 (20일 평균)(원)` | `turnover_avg_20d` | 최근 20일 평균 거래대금 |
| `순매수대금(기관계)(5일합산)(만원)` | `inst_net_5d` | 기관 최근 5일 순매수 |
| `순매수대금(기관계)(20일합산)(만원)` | `inst_net_20d` | 기관 최근 20일 순매수 |
| `순매수대금(외국인계)(5일합산)(만원)` | `foreign_net_5d` | 외국인 최근 5일 순매수 |
| `순매수대금(외국인계)(20일합산)(만원)` | `foreign_net_20d` | 외국인 최근 20일 순매수 |

> 수급 데이터는 만원 단위, 거래대금 데이터는 원 단위로 들어온다고 가정합니다.

### ③ 당일 가격·거래량 데이터

| 수집 방식 | 역할 | 비고 |
|-----------|------|------|
| FinanceDataReader | 장마감 후 당일 종가·거래량 조회 | 종목코드 6자리 기준으로 최근 OHLCV 수집 |

FinanceDataReader에서 엑셀 기준일보다 최신 거래일이 확인되면, 해당 날짜의 종가와 거래량을 가격 기준 데이터로 사용합니다. 당일 거래대금은 아래 방식으로 계산합니다.

```
당일 거래대금 = 당일 종가 × 당일 거래량
```

FinanceDataReader 최신 거래일이 엑셀 기준일보다 늦지 않으면 해당 종목은 엑셀 기준 가격 데이터를 그대로 사용합니다. 이 경우 결과 파일의 `당일가격반영` 컬럼이 `False`로 표시됩니다.

## 파생 지표 계산

입력 파일에 있는 원천 데이터를 바탕으로 아래 지표를 계산합니다.

| 지표 | 계산 방식 | 의미 |
|------|-----------|------|
| 5일 수익률 | 현재 종가 / 5거래일 전 종가 - 1 | 단기 가격 모멘텀 |
| 20일 수익률 | 현재 종가 / 20거래일 전 종가 - 1 | 중기 가격 모멘텀 |
| 5일선 | 최근 5일 종가 평균 | 단기 추세 |
| 10일선 | 최근 10일 종가 평균 | 단기 기준선 |
| 20일선 | 최근 20일 종가 평균 | 중기 기준선 |
| 기관 5일 순매수 비율 | 기관 5일 순매수 / 5일 거래대금 합계 | 거래 규모 대비 기관 매수 강도 |
| 외국인 5일 순매수 비율 | 외국인 5일 순매수 / 5일 거래대금 합계 | 거래 규모 대비 외국인 매수 강도 |
| 기관 20일 순매수 비율 | 기관 20일 순매수 / 20일 거래대금 합계 | 중기 기관 수급 강도 |
| 외국인 20일 순매수 비율 | 외국인 20일 순매수 / 20일 거래대금 합계 | 중기 외국인 수급 강도 |
| 당일 거래량 | FinanceDataReader 최신 거래일의 거래량 | 장마감 후 실제 거래량 |
| 당일 거래대금 | 당일 종가 × 당일 거래량 | 장마감 후 시장 관심도 |
| 거래대금 RVOL | 당일 거래대금 / 20일 평균 거래대금 | 당일 거래대금 증가율 |

수급 비율 계산:

```
기관 5일 순매수 비율 = 기관5일순매수(만원) × 10,000 ÷ (거래대금5일평균(원) × 5) × 100

외국인 20일 순매수 비율 = 외국인20일순매수(만원) × 10,000 ÷ (거래대금20일평균(원) × 20) × 100
```

## 종합 점수 산출 방식

세 가지 평가 영역을 가중 합산하여 0~100점 종합 점수를 부여합니다.

```
종합 점수 = 가격 (35%) + 수급 (45%) + 거래대금 (20%)
```

각 세부 항목은 전종목 횡단면 백분위 점수로 정규화합니다. 즉, 절대값 자체보다 같은 날 전체 종목 중 상대적으로 얼마나 강한지를 평가합니다.

### 가격 점수 (35%)

| 항목 | 가중치 | 의미 |
|------|--------|------|
| 20일 수익률 | 40% | 최근 한 달 가격 모멘텀 |
| 5일 수익률 | 30% | 단기 가격 모멘텀 |
| 10일선 상회 여부 | 20% | 현재 단기 추세가 우상향인지 |
| 5일선 > 10일선 여부 | 10% | 단기 이동평균 정배열 여부 |

> 5일/20일 수익률 백분위 점수는 극단값 영향을 줄이기 위해 2~98점 범위로 제한합니다.

### 수급 점수 (45%)

| 항목 | 가중치 | 의미 |
|------|--------|------|
| 외국인 5일 순매수 비율 | 25% | 단기 외국인 매수 강도 |
| 외국인 20일 순매수 비율 | 15% | 중기 외국인 매수 방향성 |
| 외국인 수급 양수 비율 | 10% | 최근 20일간 외국인 수급 지속성 |
| 기관 5일 순매수 비율 | 25% | 단기 기관 매수 강도 |
| 기관 20일 순매수 비율 | 15% | 중기 기관 매수 방향성 |
| 기관 수급 양수 비율 | 5% | 최근 20일간 기관 수급 지속성 |
| 기관·외국인 동시 순매수 | 5% | 양대 수급 주체 동시 유입 여부 |

수급은 단순 순매수 금액이 아니라 거래대금 대비 비율로 평가합니다. 대형주가 절대 금액만으로 과도하게 유리해지는 문제를 줄이기 위한 처리입니다.

### 거래대금 점수 (20%)

| 항목 | 가중치 | 의미 |
|------|--------|------|
| 거래대금 RVOL | 70% | 당일 거래대금이 최근 20일 평균 대비 얼마나 증가했는지 |
| 당일 거래대금 | 30% | 당일 유동성 및 시장 관심도 |

| 기관외인동시순매수 | 기관과 외국인이 모두 순매수인지 여부 |

## 내부 구현 상세

`whole-stock-2.py`는 실행 진입점만 담당합니다. 실제 데이터 파싱, 지표 계산, 스코어링, Excel 저장은 `score_all_supply.py`에서 처리합니다.

### ① 기본 설정

스크립트는 아래 경로와 상수를 기준으로 동작합니다.

| 항목 | 값 | 설명 |
|------|----|------|
| 입력 파일 | `C:\Users\10845\Documents\quant_project\전종목_수급.xlsx` | 원천 데이터 |
| 출력 폴더 | `C:\Users\10845\Documents\quant_project\[오후] whole-stock-2` | 결과 저장 위치 |
| 사용 시트 | `Sheet1` | 우선적으로 읽는 시트 |
| 가격 가중치 | 35% | 가격 모멘텀 반영 비중 |
| 수급 가중치 | 45% | 기관·외국인 수급 반영 비중 |
| 거래대금 가중치 | 20% | 거래대금 증가 및 유동성 반영 비중 |
| FDR 조회 기간 | 최근 80일 | 당일 가격·거래량 및 5/10/20일 지표 재계산용 |

입력 항목명은 내부 계산용 컬럼명으로 변환됩니다. 예를 들어 `종가(원)`은 `close`, `거래대금 (5일 평균)(원)`은 `turnover_avg_5d`, `순매수대금(외국인계)(5일합산)(만원)`은 `foreign_net_5d`로 매핑됩니다.

### ② 점수 정규화 방식

모든 세부 점수는 같은 날짜의 전체 종목을 대상으로 백분위 순위를 계산합니다.

예를 들어 어떤 종목의 외국인 5일 순매수 비율이 전체 종목 중 상위 10%라면 해당 항목은 약 90점대 점수를 받습니다. 반대로 하위권이면 낮은 점수를 받습니다.

정규화 방식의 목적:

- 서로 단위가 다른 지표를 0~100점 범위로 통일
- 절대 금액이 큰 대형주 편향 완화
- 매일 시장 전체 안에서 상대적으로 강한 종목을 선별

결측값은 기본적으로 50점으로 처리합니다. 해당 항목에서 정보가 부족한 종목을 과도하게 벌점 처리하지 않기 위한 중립값입니다.

### ③ 입력 시트 선택

기본적으로 `Sheet1`을 사용합니다.

만약 `Sheet1`이 없으면 파일 내 시트들을 훑어보고, 9행에 `A005930` 같은 `A` + 6자리 숫자 종목코드가 가장 많이 들어 있는 시트를 자동 선택합니다. 운영상 현재 기준은 `Sheet1` 고정입니다.

### ④ 원천 데이터 파싱

입력 엑셀은 종목별 데이터가 열 방향으로 반복되는 구조입니다. 스크립트는 이를 아래 순서로 정리합니다.

1. 9행에서 종목 코드 추출
2. 10행에서 종목명 추출
3. 13행에서 항목명 추출
4. 15행 이후 날짜별 값을 숫자로 변환
5. 종목 코드, 종목명, 날짜, 항목, 값 형태의 긴 테이블로 변환
6. 다시 날짜·종목 기준의 분석용 테이블로 피벗

최종적으로 한 행이 “특정 날짜의 특정 종목”을 의미하는 구조가 됩니다.

예시:

| date | code | name | close | turnover_avg_5d | inst_net_5d | foreign_net_5d |
|------|------|------|-------|-----------------|-------------|----------------|
| 2026-05-29 | A005930 | 삼성전자 | 종가 | 5일 평균 거래대금 | 기관 5일 순매수 | 외국인 5일 순매수 |

### ⑤ 시계열 파생 지표 계산

분석용 테이블이 만들어지면 종목별로 날짜순 정렬 후 시계열 지표를 계산합니다.

가격 관련 지표:

| 지표 | 처리 |
|------|------|
| 5일 수익률 | 종목별 종가 기준 5일 전 대비 수익률 |
| 20일 수익률 | 종목별 종가 기준 20일 전 대비 수익률 |
| 5일선 | 종가 5일 이동평균 |
| 10일선 | 종가 10일 이동평균 |
| 20일선 | 종가 20일 이동평균 |

수급 관련 지표:

| 지표 | 처리 |
|------|------|
| 기관 5일 순매수 비율 | 기관 5일 순매수를 5일 거래대금 합계로 나눈 값 |
| 외국인 5일 순매수 비율 | 외국인 5일 순매수를 5일 거래대금 합계로 나눈 값 |
| 기관 20일 순매수 비율 | 기관 20일 순매수를 20일 거래대금 합계로 나눈 값 |
| 외국인 20일 순매수 비율 | 외국인 20일 순매수를 20일 거래대금 합계로 나눈 값 |
| 기관 수급 양수 비율 | 최근 20개 관측치 중 기관 5일 순매수 비율이 양수였던 비율 |
| 외국인 수급 양수 비율 | 최근 20개 관측치 중 외국인 5일 순매수 비율이 양수였던 비율 |

거래대금 관련 지표:

| 지표 | 처리 |
|------|------|
| 거래대금 RVOL | 기본 엑셀 기준에서는 5일 평균 거래대금 / 20일 평균 거래대금 |

### ⑥ FinanceDataReader 가격 오버레이

엑셀 데이터의 최신 날짜가 수급 기준일입니다. 하지만 가격과 거래량은 장마감 후 당일 데이터를 보고 싶기 때문에, 스코어링 전에 FinanceDataReader로 각 종목의 최근 OHLCV를 다시 조회합니다.

처리 순서:

1. 엑셀에서 파싱한 전체 종목코드 목록을 생성
2. 종목별 최근 80일 OHLCV를 FinanceDataReader로 조회
3. FinanceDataReader의 최신 거래일이 엑셀 기준일보다 최신인지 확인
4. 최신이면 엑셀의 최신 수급 행을 복사해 새 가격 기준 행 생성
5. 새 행의 종가, 거래량, 거래대금, 이동평균, 수익률, 거래대금 RVOL을 FDR 기준으로 교체
6. 수급 데이터는 엑셀 기준 값을 그대로 유지

이 구조 때문에 결과 파일에는 기준일이 두 개로 표시됩니다.

| 기준일 | 의미 |
|--------|------|
| 수급 기준일 | `전종목_수급.xlsx`의 최신 날짜 |
| 가격/거래량 기준일 | FinanceDataReader에서 확인한 최신 거래일 |

FDR 최신 거래일이 당일이면 `당일가격반영=True`가 됩니다. 일부 종목에서 FDR 데이터가 늦거나 누락되면 해당 종목은 엑셀 기준 가격 데이터를 사용하고 `당일가격반영=False`로 표시됩니다.

### ⑦ 기준일 선정

스코어링은 종목별 최신 가격 행을 대상으로 수행합니다.

예를 들어 입력 파일의 수급 기준일이 2026-05-29이고 FinanceDataReader에서 2026-06-01 가격 데이터가 확인되면, 최종 결과는 가격/거래량 기준 2026-06-01 행으로 산출됩니다. 수급은 2026-05-29 기준 값을 유지합니다.

### ⑧ 스코어 계산 순서

기준일 데이터에서 아래 플래그를 먼저 만듭니다.

| 플래그 | 의미 |
|--------|------|
| `10일선상회` | 기준일 종가가 10일선보다 높은지 |
| `5일선>10일선` | 5일선이 10일선보다 높은지 |
| `기관외인동시순매수` | 기관 5일 순매수 비율과 외국인 5일 순매수 비율이 모두 양수인지 |

이후 가격스코어, 수급스코어, 거래대금스코어를 각각 계산하고 최종 종합스코어를 산출합니다.

종합스코어 산식:

```
종합스코어 = 가격스코어 × 35% + 수급스코어 × 45% + 거래대금스코어 × 20%
```

등급은 종합스코어를 기준으로 A~F를 부여합니다.

### ⑨ 결과 파일 생성

최종 결과는 Excel 파일 하나로 저장됩니다.

Excel 저장 시에는 단순 데이터 시트뿐 아니라 첫 번째 시트에 보고서 형태의 요약 화면을 생성합니다.

보고서 시트 구성:

- 제목 및 기준일
- 전체 종목 수
- A등급 종목 수
- 65점 이상 종목 수
- 80점 이상 종목 수
- 당일 가격 반영 종목 수
- 추천종목 TOP 테이블
- A+B / C / D+F 등급 분포 차트

데이터 시트에는 필터, 고정 헤더, 컬럼 폭 조정, 기본 테두리와 색상을 적용해 바로 확인할 수 있도록 정리합니다.

### ⑩ 콘솔 출력

실행이 끝나면 콘솔에는 TOP 20 종목이 출력됩니다.

출력 항목:

| 항목 | 설명 |
|------|------|
| date | 기준일 |
| code | 종목 코드 |
| name | 종목명 |
| total_score | 종합스코어 |
| grade | 등급 |
| price_score | 가격스코어 |
| supply_score | 수급스코어 |
| value_score | 거래대금스코어 |

마지막에는 저장된 Excel 경로를 출력합니다.

## 운영 유의사항

- 이 도구는 Bloomberg Terminal을 사용하지 않습니다. 단, 입력 파일 `전종목_수급.xlsx`가 별도로 최신화되어 있어야 합니다.
- 현재는 `Sheet1`을 고정으로 읽습니다. 시트명이 바뀌면 `score_all_supply.py`의 `SOURCE_SHEET` 값을 수정해야 합니다.
- 결과 파일이 Excel에서 열려 있으면 저장 실패가 발생할 수 있습니다.
- 수급 데이터는 만원 단위, 거래대금은 원 단위로 계산합니다. 입력 파일 단위가 바뀌면 수급 비율이 왜곡됩니다.
- 입력 파일의 최신 날짜는 수급 기준일로 사용됩니다. 가격/거래량 기준일은 FinanceDataReader 최신 거래일을 따릅니다.
- 종합스코어는 같은 날 종목 간 상대 백분위 기반입니다. 날짜가 다른 결과 파일끼리 점수를 직접 비교할 때는 시장 환경 차이를 고려해야 합니다.
