## 1. 도구 개요

이 도구는 **Bloomberg Terminal에서 실시간 시세 데이터를 받아** 종목의 단기 추세 전환을 자동으로 감지합니다. 매일 두 가지 용도로 사용됩니다.

| 구분 | 도구 | 용도 | 사용 시점 |
|------|------|------|-----------|
| **도구 A** | `start_bloomberg.py` | 펀드 **보유 종목** 모니터링 | 매일 오전 (장 중/장 후) |
| **도구 B** | `whole-stock.py` | **전체 시장** 스크리닝 | 매일 오후 (분석 후 보고용) |

> **공통 조건**: Bloomberg Terminal이 실행 중이고 로그인된 상태여야 합니다.

## 2. 핵심 지표 설명

### 2.1 10일선 (10일 이동평균선)

최근 10거래일의 종가 평균입니다. 단기 추세를 판단하는 기준선으로 사용합니다.

- **10일선 위** → 단기 상승 추세
- **10일선 아래** → 단기 하락 추세

### 2.2 10일선 돌파일 / 이탈일

| 용어 | 의미 | 해석 |
|------|------|------|
| **10일선 돌파일** | 종가가 10일선을 **위로 뚫은** 날짜 | 상승 추세 전환 시점 |
| **10일선 이탈일** | 종가가 10일선을 **아래로 뚫은** 날짜 | 하락 추세 전환 시점 |

> 이탈일이 오래될수록 하락 추세가 지속되고 있음을 의미합니다. 돌파일이 이탈일보다 최근이면 현재 10일선 위에 있는 것입니다.

### 2.3 RVOL (Relative Volume, 상대 거래량)

```
RVOL = 현재 거래량 ÷ 최근 10일 평균 거래량
```

평소 거래량 대비 오늘 거래량이 몇 배인지를 나타냅니다.

| RVOL 값 | 의미 |
|---------|------|
| 1.0 미만 | 거래 한산 |
| 1.0 ~ 1.5 | 보통 수준 |
| **1.5 이상** | **거래량 유의미한 증가 → 신호 발생 기준** |
| 2.0 이상 | 거래량 폭증 |

### 2.4 10일선 괴리율

```
괴리율 = (현재가 - 10일선) ÷ 10일선 × 100%
```

양수(+)면 10일선 위, 음수(-)면 10일선 아래에 있으며, 절댓값이 클수록 추세가 강합니다.

### 2.5 RSI (상대강도지수)

0~100 범위. 70 이상은 과매수, 30 이하는 과매도 구간입니다.

## 3. 신호 해석 가이드

### 3.1 도구 A (보유 종목) — 3가지 분류

| 분류 | 조건 | 의미 | 대응 |
|------|------|------|------|
| 🔴 **강력 매도 신호** | 10일선 **아래** + RVOL ≥ 1.5 | 하락 추세 확인 + 매도세 집중 | 익절 검토 우선순위 1순위 |
| 🟡 **profit taking** | 10일선 **아래** + RVOL < 1.5 | 하락 추세이나 아직 거래량 미약 | 모니터링 강화, 거래량 증가 시 대응 |
| 🟢 **upside** | 10일선 **위** + RVOL ≥ 1.5 | 상승 추세 + 거래량 동반 상승 | 모멘텀 유지 확인 |

**추가 필터 조건** (강력 매도 신호 / profit taking에만 적용):
- 10일선 이탈일이 **최근 5영업일 이내**인 종목만 표시
- 10일선 돌파일과 이탈일의 차이가 **3일 이상**이어야 유효한 신호로 판단
  - (단순 노이즈로 인한 일시적 이탈 제거)

### 3.2 도구 B (전종목 스크리닝) — 2가지 유형

| 유형 | 조건 | 의미 |
|------|------|------|
| **타입 A** | 10일선 돌파일 = **오늘** + RVOL ≥ 1.5 | 오늘 10일선을 돌파한 종목 (모멘텀 진입 후보) |
| **타입 B** | 10일선 이탈일 = **오늘** | 오늘 10일선을 이탈한 종목 (하락 전환 감시) |

## 4. 도구 A — 보유 종목 모니터링

### 목적

펀드에서 **현재 보유 중인 종목들**의 단기 추세를 매일 점검하고, 익절 타이밍을 판단하기 위한 도구입니다.

### 입력

사용자가 직접 Bloomberg 티커를 입력합니다.

```
예시: 005930 KS, 000660 KS, 373220 KS, AAPL US, MSFT US
```

제외할 티커도 선택적으로 입력할 수 있습니다 (이미 매도한 종목 등).

### 분석 모드

| 모드 | 설명 | 사용 시점 |
|------|------|-----------|
| **[1] 보고용** | 완성된 일봉 기준 (장 중에는 전일까지, 장마감 후에는 당일 포함) | 오전 보고, 장마감 후 정리 |
| **[2] 실시간** | 현재 시점의 미완성 일봉 포함 | 장 중 즉시 확인 필요 시 |

### 데이터 흐름

```
사용자 티커 입력
       ↓
Bloomberg API → 최근 3개월 OHLCV 데이터 수신 (10개 병렬)
       ↓
10일선 위치 판단 + RVOL 계산 + RSI 계산
       ↓
3가지 분류: 강력 매도 신호 / profit taking / upside
       ↓
필터링: 이탈일 최근 5일 이내 + 돌파-이탈 간격 3일 이상
       ↓
Excel 저장 + 히트맵 생성 (선택)
```

### 출력

**엑셀 파일** → `C:\Users\Bloomberg\Documents\ssh_project\[오전] start-bloomberg-result\`
```
조건별_분류_보고용_20260225_091500.xlsx
```

| 시트 | 내용 |
|------|------|
| **전체** | 3개 분류 통합, 카테고리 순서대로 정렬 |
| **강력매도신호** | 🔴 분류 전용 시트 |
| **profit taking** | 🟡 분류 전용 시트 |
| **upside** | 🟢 분류 전용 시트 |

**히트맵** → `database\YYYYMMDD_HHMMSS_heatmap.png`
- 종목별 추세 및 신호 분포를 시각화한 이미지

## 5. 도구 B — 전종목 스크리닝

### 목적

코스피·코스닥 전종목 및 주요 ETF 약 **3,000개 이상**을 대상으로 오늘 10일선을 돌파하거나 이탈한 종목을 자동으로 찾아냅니다. 새로운 진입 기회 또는 위험 신호를 시장 전체에서 탐색합니다.

### 분석 대상

분석 대상은 `bloomberg_ticker.xlsx` 파일에서 자동으로 불러옵니다.

| 시트 | 내용 | 종목 수 |
|------|------|---------|
| **Sheet1** | 코스피 + 코스닥 전종목 | 약 2,900개 |
| **해외ETF** | 미국 상장 주요 ETF (SPY, QQQ, SMH 등) | 41개 |
| **한국ETF** | 국내 상장 주요 ETF (KODEX, TIGER 등) | 28개 |

**해외ETF 분류:**
미국지수 (SPY/QQQ/IWM/DIA), 섹터 (XLK/XLF/XLE 등), 채권 (TLT/IEF/HYG), 원자재 (GLD/SLV/GDX), 레버리지 (TQQQ/SOXL 등), 개별레버리지 (NVDL/TSLL 등), 변동성 (VXX/VIXY), 크립토 (IBIT) 등

**한국ETF 분류:**
국내지수 (KODEX 200/레버리지/인버스), 국내섹터 (반도체/방산/조선 등), 해외추종 (TIGER 미국S&P500/나스닥100 등) 등

### 스크리닝 유형 선택

실행 시 두 가지 유형 중 선택합니다:

**타입 A — 10일선 돌파 종목**
- 오늘 10일선을 돌파한 종목 + RVOL ≥ 1.5
- "오늘 새롭게 모멘텀이 생긴 종목은 무엇인가?"
- 정렬: 시가총액 내림차순 → 10일선 이탈일 오름차순

**타입 B — 10일선 이탈 종목**
- 오늘 10일선을 이탈한 종목 (RVOL 무관)
- "오늘 추세가 꺾인 종목은 무엇인가?"
- 정렬: 시가총액 내림차순

### 분석 모드

도구 A와 동일한 보고용 / 실시간 모드를 지원합니다.

### 데이터 흐름

```
bloomberg_ticker.xlsx 전체 시트 읽기 (~3,000개 티커)
       ↓
Bloomberg API → 최근 3개월 OHLCV 병렬 수신 (15개 동시)
       ↓
종목별: 10일선 위치, RVOL, RSI, 돌파일/이탈일 계산
       ↓
[타입 A] 10일선 위 + RVOL≥1.5 + 돌파일=오늘 필터링
[타입 B] 10일선 아래 + 이탈일=오늘 필터링
       ↓
시가총액 조회 → 정렬 → Excel 저장
```

### 출력

**엑셀 파일** → `C:\Users\Bloomberg\Documents\ssh_project\[오후] whole-stock-result\`

| 파일명 | 조건 |
|--------|------|
| `전종목_스크리닝_보고용_YYYYMMDD_HHMMSS.xlsx` | 타입 A + 보고용 |
| `전종목_스크리닝_실시간_YYYYMMDD_HHMMSS.xlsx` | 타입 A + 실시간 |
| `10일선하회_보고용_YYYYMMDD_HHMMSS.xlsx` | 타입 B + 보고용 |
| `10일선하회_실시간_YYYYMMDD_HHMMSS.xlsx` | 타입 B + 실시간 |

| 시트 | 내용 |
|------|------|
| **스크리닝결과** | 오늘 조건을 충족한 종목 (한국 주식) |
| **ETF현황** | 전체 ETF 분석 결과 (날짜 필터 없음, 10일선 상태·RVOL 기준 정렬) |

## 6. 출력 파일 상세

### 컬럼 설명

| 컬럼명 | 설명 |
|--------|------|
| **종목명** | 한글 종목명 (Bloomberg에서 조회) |
| **티커** | Bloomberg 티커 (예: `005930 KS`, `SPY US`) |
| **시가총액** | Bloomberg 기준 현재 시가총액 |
| **카테고리** | 강력 매도 신호 / profit taking / upside |
| **10일선상태** | ETF현황 시트 전용. `10일선 위` 또는 `10일선 아래` |
| **RVOL** | 상대 거래량 (배수) |
| **RSI** | 상대강도지수 (0~100) |
| **10일선돌파일** | 가장 최근 10일선을 위로 돌파한 날짜 |
| **10일선이탈일** | 가장 최근 10일선을 아래로 이탈한 날짜 |
| **현재가** | 분석 기준 종가 |
| **전일비(%)** | 전일 대비 등락률 |
| **10일선** | 10일 이동평균 가격 |
| **10일선괴리율(%)** | 현재가와 10일선의 괴리율 (+ 는 위, - 는 아래) |
| **추세상세** | 추세 전환 상세 내용 (예: `10일선 아래(01-20) → 10일선 위(02-05)`) |

## 8. 실행 방법

### 도구 A — 보유 종목 모니터링

```bash
py -3.12 start_bloomberg.py
```

실행 후 순서:
1. Bloomberg 티커 입력 (쉼표 구분): `005930 KS, 000660 KS, AAPL US`
2. 제외 티커 입력 (선택, 없으면 엔터)
3. 모드 선택: `1` (보고용) 또는 `2` (실시간)
4. 분석 완료 후 히트맵 저장 여부: `y/n`
5. Excel 저장 여부: `y/n`

### 도구 B — 전종목 스크리닝

```bash
py -3.12 whole-stock.py
```

실행 후 순서:
1. 스크리닝 유형 선택: `A` (10일선 돌파) 또는 `B` (10일선 이탈)
2. 모드 선택: `1` (보고용) 또는 `2` (실시간)
3. 자동으로 bloomberg_ticker.xlsx 전체 시트 로드 → 분석 시작
4. 완료 후 자동 Excel 저장

### ETF 종목 추가/수정

`bloomberg_ticker.xlsx` 파일의 **해외ETF** 또는 **한국ETF** 시트에서 직접 추가하거나,
`add_etf_sheets.py` 스크립트를 수정 후 재실행합니다.

```bash
py -3.12 add_etf_sheets.py
```
